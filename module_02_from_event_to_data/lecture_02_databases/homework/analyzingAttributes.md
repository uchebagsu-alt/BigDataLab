# Анализ атрибутов 
Атомарность - в одной ячейке таблицы должно лежать только одно неделимое значение.  
Шаги проверки на атомарность таблицы:
1) нет повторяющихся групп(атрибутов) (например, id1, id2, id3)
2) нет списков в ячейках (например, 'Вася;Петя;Славик' - НЕ АТОМАРНО, 'Вася' - АТОМАРНО)
3) нет несколько типов (что-то одно: одна строка, одно число и так далее)

## products
**Атрибуты**:

- *product_id* - id продукта, хранит уникальные числа-индентификаторы
- product_name - Название продукта, нету списков в ячейках и разных значений
- price - цена товара, мы не можем число в ячейке разделить по делителю не меняя смысл, одно только значение в ячейках
- category_id - id категории товара, одно значение - число, и не повторяет смысл с product_id 
- class - Класс продукта, только одно значение в каждой ячейке
- modify_timestamp - хранит тип TIMESTAMP, т.е. одно значение - время
- resistant - хранит значение есть ли усточивость продукта, по одному значение в каждой ячейке No или Yes
- is_allergic - хранит значение есть ли аллергены в продукте, так что не повторяет смысл resistant, в ячейках тоже No или YES
- vitality_days - Срок годности продукта, нет разделителей,

<small>Итог: нет повторяющихся групп, нет списков в ячейках и нет несколько типов - атомарно</small>

## employees
**Атрибуты**:

- employee_id - id сотрудника, хранит уникальные числа-индентификаторы
- first_name - хранит только Имя сотрудника
- middle_initial - хранит только инициал сотрудника - букву
- last_name - хранит только последнее Имя
- birth_date - тип DATETIME(дата рождения)
- gender - гендер сотрудника, только одно значение.
- city_id - id города, нет списков
- shop_id - id магазина, нет списков
- hire_date - дата найма на работу, иной смысл атрибута, чем с birth_date

<small>Итог: нет повторяющихся групп, нет списков в ячейках и нет несколько типов - атомарно</small>
