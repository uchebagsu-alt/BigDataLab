# Ограничения (Constraints) для sales и products

## Ограничения NOT NULL (поля должны обязательно быть заполнены)
- *products.product_name* : товар без названия не может быть.
- *sales.sales_timestamp* : дата и время продажи товара очень важны. Нельзя провести продажу без фиксации времени.
- *sales.product_id* : продажа не может быть без указания проданного товара.
- *sales.total_price* : операция не имеет смысла без итоговой суммы (даже, если она равна нулю при 100% скидке, значение должно быть указано).

## Ограничения UNIQUE (Уникальность)
- *products.product_id* : идентификатор товара должен быть уникальным.
- *sales.transaction_number* : номер чека или транзакции должен быть уникальным во избежание дублирования.

## Ограничения CHECK (Проверка значений)
- *sales.total_price >= 0* : итоговая цена продажи не может быть отрицательной.
- *sales.quantity > 0* : количество проданного товара должно быть только положительным числом и больше 0.
- *sales.discount BETWEEN 0 AND 1* : 0 - без скидки, 1 - 100% скидка.
